  <action>
    Scaffold a new Vite project and install all dependencies. Follow the exact installation sequence from RESEARCH.md to avoid the shadcn init-before-config pitfall:

    1. Run `npm create vite@latest . -- --template react-ts` to scaffold in the current directory (project is greenfield). If the directory has planning files, use a temp directory and move files, or scaffold with a project name and move contents up.

    2. Run `npm install` to install base dependencies.

    3. Install Tailwind CSS v4: `npm install tailwindcss @tailwindcss/vite`

    4. Install React Router v7: `npm install react-router` (NOT react-router-dom -- that's v6).

    5. Install font and icons: `npm install @fontsource-variable/inter lucide-react`

    6. Install Node types for path alias: `npm install -D @types/node`

    7. Configure `vite.config.ts` with three plugins/settings:
       - `@vitejs/plugin-react` (already present from scaffold)
       - `@tailwindcss/vite` plugin: `import tailwindcss from "@tailwindcss/vite"` and add `tailwindcss()` to plugins array
       - Path alias: `resolve: { alias: { "@": path.resolve(__dirname, "./src") } }`
       - Import `path` from `"path"` at the top

    8. Configure path aliases in BOTH tsconfig files:
       - `tsconfig.json`: Add `"compilerOptions": { "baseUrl": ".", "paths": { "@/*": ["./src/*"] } }`
       - `tsconfig.app.json`: Add same `baseUrl` and `paths` to compilerOptions

    9. Set up minimal `src/index.css` with just `@import "tailwindcss";` (theme comes in Task 2).

    10. Set up `src/main.tsx` with BrowserRouter wrapper and Inter font import:
        ```typescript
        import React from "react";
        import ReactDOM from "react-dom/client";
        import { BrowserRouter } from "react-router";
        import App from "./App";
        import "@fontsource-variable/inter";
        import "./index.css";

        ReactDOM.createRoot(document.getElementById("root")!).render(
          <React.StrictMode>
            <BrowserRouter>
              <App />
            </BrowserRouter>
          </React.StrictMode>
        );
        ```

    11. Create a minimal `src/App.tsx` placeholder (just renders a div with "App" text) so the project compiles.

    12. Initialize shadcn/ui: `npx shadcn@latest init` -- select "New York" style, confirm defaults. This must happen AFTER vite.config.ts and tsconfig paths are configured.

    13. Add all needed shadcn components in one command:
        `npx shadcn@latest add button card tabs breadcrumb input label select textarea separator`

    14. Verify the dev server starts: `npm run dev` should serve on localhost without errors.

    IMPORTANT: Delete any default Vite boilerplate files (src/App.css, src/assets/react.svg, etc.) that won't be used.
  </action>

<task type="auto">
  <name>Task 2: Define forest/earth color palette and corporate theme in index.css</name>
  <files>
    src/index.css
  </files>
  <action>
    Replace the minimal index.css with the full themed version. This file is the single source of truth for all design tokens.

    Structure the file in this order:

    1. Font import: `@import url("...")` is NOT needed -- we use @fontsource-variable/inter imported in main.tsx. No Google Fonts CDN.

    2. Tailwind import: `@import "tailwindcss";`

    3. `@theme` block defining custom color scales and font:
       - `--font-sans`: `'Inter Variable', 'Inter', ui-sans-serif, system-ui, sans-serif`
       - `--color-forest-*` scale (50-950): oklch values with hue ~155, low chroma (0.01-0.10) for muted greens. Use the values from RESEARCH.md as starting point.
       - `--color-earth-*` scale (50-950): oklch values with hue ~60-80, low chroma for warm tans.
       - `--color-charcoal-*` scale (50-900): oklch values with hue ~250, very low chroma (0.005) for neutral grays.

    4. `:root` block with shadcn/ui CSS variable overrides mapped to forest/earth palette:
       - `--background`: warm off-white (earth-tinted, not pure white) -- `oklch(0.985 0.005 80)`
       - `--foreground`: deep charcoal -- `oklch(0.16 0.005 250)`
       - `--card`: pure white `oklch(1.0 0 0)` for card surfaces
       - `--card-foreground`: same as foreground
       - `--primary`: forest-700 `oklch(0.35 0.08 155)` -- deep forest green as the brand accent
       - `--primary-foreground`: light warm white for text on primary
       - `--secondary`: earth-100 `oklch(0.93 0.02 80)` -- light tan for secondary surfaces
       - `--secondary-foreground`: dark charcoal
       - `--muted`: earth-50 area for muted backgrounds
       - `--muted-foreground`: mid-gray for secondary text
       - `--accent`: forest-100 `oklch(0.93 0.02 155)` -- light green for hover/focus accents
       - `--accent-foreground`: dark green
       - `--destructive`: warm red `oklch(0.55 0.2 25)` for error states
       - `--border`: light earth-tinted border
       - `--input`: same as border
       - `--ring`: forest-400 for focus rings
       - `--radius`: 0.5rem

    5. `@theme inline` block bridging shadcn CSS variables to Tailwind utility classes:
       ```css
       @theme inline {
         --color-background: var(--background);
         --color-foreground: var(--foreground);
         --color-card: var(--card);
         --color-card-foreground: var(--card-foreground);
         --color-primary: var(--primary);
         --color-primary-foreground: var(--primary-foreground);
         --color-secondary: var(--secondary);
         --color-secondary-foreground: var(--secondary-foreground);
         --color-muted: var(--muted);
         --color-muted-foreground: var(--muted-foreground);
         --color-accent: var(--accent);
         --color-accent-foreground: var(--accent-foreground);
         --color-destructive: var(--destructive);
         --color-border: var(--border);
         --color-input: var(--input);
         --color-ring: var(--ring);
       }
       ```

    6. Add a base body style reset:
       ```css
       body {
         font-family: var(--font-sans), sans-serif;
         background-color: var(--background);
         color: var(--foreground);
       }
       ```

    CRITICAL: Use `@import "tailwindcss"` (v4 syntax), NOT `@tailwind base; @tailwind components; @tailwind utilities;` (v3 syntax). Do NOT create a tailwind.config.js file.

    Reference the exact oklch values from RESEARCH.md Pattern 2. The palette should feel: muted, grounded, professional -- think consulting firm environmental report, not bright eco-marketing.
  </action>
